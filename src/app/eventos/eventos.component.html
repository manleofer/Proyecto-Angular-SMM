EVENTOS


Definición:
Un evento es una acción o suceso que ocurre en el navegador y que es detonado por el usuario.
Por ejemplo: hacer clic en un botón, mover el puntero del ratón, presionar una tecla...
Angular permite capturar estos eventos y ejecutar funciones en respuesta, mediante la técnica "event binding"

Event Binding:
Consiste en la vinculación de eventos: Asocia eventos del DOM con métodos definidos en el componente. Se realiza con la sintaxis básica:
<elementoHTML (evento)="método($event)""></elementoHTML>


Algunos eventos en Angular:

  • Eventos del Ratón

    • (click) → Cuando se hace clic

      • Ejemplo: Al hacer click en el siguiente botón, éste alterna su estilo de bootstrap

    • (dbclick) → Cuando se hace doble clic
    • (mouseenter) → Cuando el ratón entra en un área (no se propaga a los elementos hijos)
    • (mouseover) → Cuando el ratón entra en un área (se propaga a los elementos hijos)
    • (mouseleave) → Cuando el ratón sale de un área

      • Ejemplo 1: A continuación se muestran un conjunto de recuadros anidados, uno dentro de otro. Haciendo uso de (mouseenter), al entrar en el recuadro padre, éste cambia de color. Sin embargo, al entrar en los recuadros descendientes (anidados), el color no cambia. Es decir, el evento no se propaga desde el padre hacia sus descendientes. Haciendo uso de (mouseleave), el color del recuadro padre se restaura al salir al exterior del mismo.

      Ejemplo 2: A continuación se muestra el mismo conjunto de conjunto de recuadros, pero se hace uso de (mouseover). En este caso, al entrar en los elementos anidados, ellos también cambian su color. Es decir, el evento se propaga desde el padre hacia sus descendientes. Esta vez, haciendo uso de (mouseleave), el color de cada recuadro se restaura al salir del mismo.

    • (keydown) → Cuando se pulsa una tecla

    • (keypress) → Cuando se mantiene presionada una tecla

      • Ejemplo: Al escribir en el primer recuadro de texto, el mensaje se reproduce a tiempo real en el segundo recuadro, inicialmente vacío. Todo ello ocurre al pulsar cada una de las teclas, haciendo uso de (keydown)

  • Eventos de Entrada
    • (input) → Cuando el usuario escribe
    • (change) → Cuando se cambia el valor o se pierde el foco
    • (focus) → Cuando se recibe el foco
    • (blur) → Cuando se pierde el foco

      • Ejemplo: Al escribir en el input aparece un mensaje indicando tal acción. Se usa (input). Al cambiar el foco entre los distintos elementos, aparece un mensaje indicando tal acción. Se usa (focus). Al cambiar entre las opciones del select, aparece un mensaje indicando tal acción. Se usa (change).

  • Eventos de Formularios
    • (submit) → Cuando se envía el formulario
    • (reset) → Cuando se restablece el formulario

      • Ejemplo: En el apartado de "Formularios", se maneja el evento (reset), apareciendo un mensaje indicando que el formulario se ha restablecido.


Angular maneja muchos eventos más, aunque no son los más habituales. Para más información, consultar la documentación oficial de Angular.
